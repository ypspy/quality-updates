# SUMMARIZE 경제성 진단 — Design

**날짜**: 2026-07-23  
**상태**: 승인된 설계 (진단 실행·슬림 구현은 후속)  
**대상**: `.claude/skills/quality-updates-writer` — **SUMMARIZE**만 (BOILERPLATE·SKIP_REMOVAL 제외)

---

## 1. 목적

Cursor에서 Phase 3 SUMMARIZE가 **고정·반복 컨텍스트로 비싸게 설계됐는지**를 분해하고, **현재 gold 수준(어조·상세도 포함)을 유지**하면서 줄일 수 있는 항목만 우선순위 백로그로 정리한다.

이번 사이클은 **진단만** 한다. SKILL·gold·분기 본문 변경은 하지 않으며, 슬림 후보는 **별도 승인** 후에만 구현 스펙으로 넘긴다.

로컬 LLM 도입·재논의는 범위 밖이다.

| 항목 | 기준 |
|------|------|

절감폭이 작아도 품질을 깎는 후보는 채택하지 않는다.

## 3. 표본 코퍼스 (실측·규모 추정)

요약 직전 상태인 분기를 실측·규모 추정의 **정본 표본**으로 쓴다.

| 항목 | 값 |
|------|-----|
| 파일 | `docs/quality-updates/2016/2016-01-01_to_2016-03-31.md` |
| 스냅샷(설계 시점) | `!!! note` 0 · `<!-- source: -->` **46** (pdf 23 + shot 23) · `<!-- skip -->` ~545 · `<!-- no_summary -->` 13 · 파일 ~272KB |
| 용도 | (1) 분기 규모·고정비 대비 가변비 추정 (2) 실측 3~5링크 선정 풀 |

**실측 링크 선정 규칙** (동일 파일에서):

1. 일반 `!!! note` 후보 — 짧은 pdf source 1~2건  
2. 긴·수치 많은 pdf 1건 (예: 외감/재무제표·감리 계열)  
3. `shot` source 1건 (웹/캡처 경로 비용)  
4. Type A/B(제재 표)가 큐에 있으면 가능하면 1건 포함; 없으면 일반 note로 대체하고 표는 정적 분석만

실측 중에는 **현행 SUMMARIZE 규칙 그대로** 돌린다. 진단 목적의 note 초안은 HITL이 배포용으로 쓰지 않아도 되며, 본문 반영은 사용자 지시가 있을 때만 한다(기본: 실측 로그·초안은 워크트리/임시 기록, 2016 파일 대량 커밋 금지).

## 4. 접근: 비용 분해 감사

고정비 / 링크 가변비 / 재작업비로 나누고, 항목마다 아래를 채운다.

| 열 | 의미 |
|----|------|
| 추정 토큰 | 하한~상한 (바이트÷4 기본, 한국어 보수 ÷3 병기) |
| gold 기여 | 필수 / 보조 / 의심 |
| 슬림 가설 | 한 줄 (예: gold 발췌만, 분기 md 윈도우만) |
| 품질 위험 (C) | 낮 / 중 / 고 + 한 줄 근거 |
| 우선순위 | `절감_추정 × (1 − 위험가중)` (위험 높으면 하단·보류) |

### 4.1 초안 핫스팟 목록 (정적 분석 시 반드시 채울 것)

**고정비**

- SKILL.md 전문 로드 (~17KB)  
- gold 2파일 **전문** 읽기 의무 (~192KB + ~218KB; 세션당 의도된 상한이 큼)  
- Announce · TaskCreate 체크리스트 의식  
- REFERENCE A–G를 매 링크마다 재독하는지 여부

**링크 가변비**

- PDF/`shot` 원문 로드·추출 텍스트 길이  
- 대상 분기 md: 파일 전체(~272KB) vs 링크 주변 윈도우  
- note 초안 생성·삽입  
- Type A/B 표 스키마 준수 비용

**재작업비**

- OCR/`shot` fallback  
- HITL 사실·어조 수정 라운드  
- 동일 링크 재요약

측정: **정적 맵으로 Top N → 상위 1~2 핫스팟에 걸린 2016 링크 3~5건 실측**. Cursor 대시보드 수치가 있으면 병기; 없으면 상대 비중(%)로 충분. Cursor $ 단가의 정확한 역산은 필수 아님.

## 6. 판정 규칙

| 판정 | 조건 |
|------|------|
| **유지** | gold 기여=필수이거나, 절감 대비 C 위험 큼 |
| **슬림 후보** | 추정 절감이 의미 있고, C 위험=낮~중이며 완화책 있음 → **구현은 별도 승인** |
| **보류** | 실측 부족·위험 불명·절감 미미 |

## 8. 워크플로

```
정적: SKILL + gold + 2016 Q1 규모 맵
  → 분해표 초안 + 핫스팟 Top N
  → 2016 Q1에서 3~5링크 현행 SUMMARIZE 실측
  → 표·백로그·판정 확정 (본 스펙에 기입)
  → (별도 승인) 슬림 항목만 구현 스펙
```

구현 계획은 본 스펙 사용자 검토 후 `writing-plans`로 작성한다.
