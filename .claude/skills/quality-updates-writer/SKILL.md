# 에이전트 지침: Quality Updates – 회계·감사 규제 동향 요약

> **기준 파일(gold standard)**: `2024-10-01_to_2024-12-31.md`, `2025-10-01_to_2025-12-31.md` — 포맷·어조·상세 수준의 기준. 판단이 어려우면 이 파일들을 참조할 것.

## 최우선 원칙
- **추정 절대 금지**: 원문(PDF/HWP) 또는 큐레이터 미리보기(iframe)에서 **직접 확인되지 않은 내용**은 요약에 포함하지 않는다.
- 원문 접근이 느리거나 일시적으로 비어 보이면, 큐레이터에서 **기다리기/미리보기 새로고침 등 재시도**를 우선 수행한다.
- 재시도 후에도 확인 불가이면 `<!-- 원문 접근 불가 -->` 처리(또는 해당 항목을 **미결정**으로 유지)한다.

## 역할

한국 회계·금융 당국(금융감독원, 금융위원회, 한국공인회계사회, 한국회계기준원)의 보도자료·공지사항·규정을 읽고, **MkDocs 기반** 구조화된 요약을 생성하는 에이전트.

## 큐레이션 편집기·링크 상태

- 로컬에서 `python scripts/editor.py`로 분기 `.md`를 연다. (의존성: `pip install flask`). 기술·동작 요약: **`docs/editor-curation-workflow.md`**.
- **미결정**: 링크 줄만 있고 그 아래에 `<!-- source:… -->` / 요약 블록이 없음 → **처리 전**(제목·링크만 노출 수준).
- **요약 필요**: 링크 다음에 `<!-- source: pdf|… -->`(또는 레거시 `<!-- pdf: … -->`), `web`, `clip` 등 → Phase 0·1의 **요약·원문 연결 큐**.
- **요약 없음(링크만)**: 링크는 남기되 요약 작업에서 제외 → 링크 다음 줄에 `<!-- no_summary -->`로 기록. 에이전트는 해당 링크를 **요약/처리하지 않는다.**
- **스킵**: 링크 다음 줄에 `<!-- skip -->` → 편집기 저장 시 동일하게 기록. MkDocs만으로는 링크 불릿은 남고 주석만 숨겨지므로, 배포본에서 항목까지 없애려면 아래 **「배포 전 스킵 제거」**를 수행한다.
- **완료**: 링크 직후 `!!! note` / `??? note` 등 — 기존 Phase 1·2 규칙대로 유지·보강.

### 배포 전 스킵 제거 (에이전트 지시 시)

공개 사이트에 스킵 항목을 남기지 않으려면, 사용자가 배포·정리 단계에서 요청할 때 다음을 따른다.

1. **대상**: `docs/quality-updates/**/*.md` 본문(특히 `## Appendix` 이전 목록 구역).
2. **식별**: 표준 링크 줄 `- (YY-MM-DD) [제목](URL)` 바로 다음(중간 빈 줄 0~1개 허용)에 **단독 줄** `<!-- skip -->`이 오는 쌍.
3. **수정**: 위 **링크 줄과 `<!-- skip -->` 줄을 함께 삭제**한다. 앞뒤 리스트가 끊기지 않게 빈 줄만 정리한다.
4. **제외**: `<!-- skip -->`이 없는 링크만 있는 행은 **미결정**이므로 삭제하지 않는다.

## 작업 순서 (필수)

| 단계 | 설명 |
|------|------|
| **Phase 0** | 각 링크 처리 전, `downloads/` 폴더에서 매칭 PDF 검색 → 있으면 `extract_pdf.py` 실행 후 Phase 1 진행 |
| **Phase 1** | 각 링크의 원문(PDF 추출물 또는 웹)을 읽고 `!!! note` / `??? note` 요약 블록 생성 |
| **Phase 2** | Phase 1 결과를 토대로 Executive Summary·기관별 요약·시사점 생성 |

### WEB 요약 시(링크 미리보기 기반)
- **출처가 `web`인 링크**는, 큐레이션 편집기에서 해당 링크의 **WEB 미리보기 화면(iframe)**에 표시되는 전처리/정제된 내용을 원문 근거로 삼아 요약한다.
- WEB 접근이 불안정할 수 있으므로, 큐레이터에서 **미리보기 캡쳐(스크린샷) 저장**을 수행해 `<!-- source: shot|downloads/...png -->` 형태로 **증거(표/레이아웃)를 고정**한 뒤 요약에 사용한다. (필요 시 OCR 결과 CLIP도 함께 생성됨)
- 미리보기에서 확인 가능한 **핵심 문단/시행(적용) 시점/수치/조치 요지/담당 주체**를 중심으로 Phase 1 요약 블록 불릿을 작성하고, **URL/원문 원문 문장을 그대로 재현하지 않는다.**
- 미리보기 화면이 비어있거나 핵심 정보가 부족한 경우에는 **먼저 추정하지 말고**, 큐레이터에서 **잠시 기다리며 재로딩/새로고침**을 시도한다.
- 재시도 후에도 확인 불가이면 `<!-- 원문 접근 불가 -->` 처리(또는 `미결정` 상태 유지) 후 다음 링크로 넘어간다.

## Phase 0: PDF 경로 결정

아래 순서로 적용. 결정되면 `scripts/pdf_path.txt`에 UTF-8로 경로 저장 → `scripts/extract_pdf.py` 실행 → `scripts/pdf_extracted.txt` 사용.

| 우선순위 | 조건 | 동작 |
|----------|------|------|
| 1 | 사용자가 경로 지정 | 해당 경로 사용 |
| 2 | 요약 지시 시 (경로 미지정) | `downloads/*.pdf`에서 링크 제목 키워드 포함 파일 검색 → 매칭 시 사용 |
| 3 | "관련 PDF" 요청 | `downloads/1.pdf` 사용. 없으면 경로 확인 요청 |
| 4 | "최근 PDF" 요청 | `downloads/` 폴더 수정일 내림차순 첫 번째 파일 사용. 0건이면 경로 요청 |

### 스크립트 사용

`scripts/extract_pdf.py` 사용. 경로는 `scripts/pdf_path.txt`에 UTF-8로 한 줄 저장 후 실행. 출력: `scripts/pdf_extracted.txt`. `pypdf` 미설치 시 `pip install pypdf`. 블록 안에 PDF URL 불포함.

### 경로 인코딩 이슈

파일명에 날짜 따옴표(`'25.8.27.`) 등 특수 유니코드(U+2019)가 있으면 `FileNotFoundError` 발생 가능. 스크립트는 실패 시 날짜 접두어(`250827*.pdf`) 패턴으로 glob 검색 후 첫 매칭 파일 사용 — 경로 그대로 저장 후 실행해도 fallback으로 추출 시도함.

### HWP 및 기타 원문

- **HWP**: 수동으로 PDF 변환 후 PDF 추출 워크플로우 사용.
- **한글 경로**: 셸 인코딩 이슈로 한글 경로 직접 전달 시 깨질 수 있음. 경로를 UTF-8 텍스트 파일에 저장 후 스크립트 path 파일 사용.

# Phase 1: 개별 링크 요약

**요약 배치**: 각 링크에 대한 요약(`!!! note` / `??? note` 블록)은 **해당 링크 줄 바로 하단**에 추가함. 링크와 요약 블록 사이에는 빈 줄 1개를 둠. 또한 요약 블록의 마지막 줄과 **다음 링크 줄 사이에도** 빈 줄 1개를 둠.

## 1.1 블록 문법·들여쓰기

| 상황 | 문법 |
|------|------|
| 일반 (불릿 3~5개) | `!!! note "주요 내용"` |
| 긴 요약 (6개↑, 표·상세 포함) | `??? note "주요 내용"` |
| 증선위 조치 의결 (Type B) | `??? note "조사·감리결과 지적사항 및 조치내역"` |
| 금융위 최종 과징금 (Type A) | `!!! note "조사·감리결과 최종 과징금 부과"` |

> `!!!`/`???` 선택은 불릿 수만이 아니라 길이·밀도를 종합 고려. 과도하게 길면 `???`.

**들여쓰기**: 요약 블록은 **링크 줄 바로 하단**에 위치. 블록은 링크 줄로부터 **4칸**, 블록 내용은 추가 **4칸**, 블록 줄과 첫 내용 사이에 **빈 줄 1개** 필수.

```markdown
- (24-12-04) [제목](https://...)

    !!! note "주요 내용"

        - 첫 번째 불릿
            - 하위 불릿 (4칸 추가)
```

## 1.2 작성 규칙

- **불릿 수**: 3~7개 (긴 문서 8~9, 짧은 공지 2~3 허용)
- **접두어**: 블록 단위로 하나만 사용 (혼용 금지)
    - **스타일 A 괄호**: 서사적 흐름 — `(배경)`, `(개요)`, `(향후)` 등 (2~6자)
    - **스타일 B 볼드-콜론**: 독립 주제 — `**감사전 재무제표**: …`
- **볼드**: 핵심 제도명·기관명·기준치·키워드에만
- **어미**: 명사형/~함/~임/~예정 (음/슴체·합니다체 금지)
- **내용 증류**: what·why·impact 중심, 수치·금액·날짜는 보존, 원문 재현 금지

## 1.3 요약 대상 선별

**요약 O**: 정책 변경·제도 개정, 감독 운영계획, 주요 간담회·설명회, 제재 조치(과징금·제재), 표준감사시간, 내부회계관리제도, 보험건전성(IFRS17·K-ICS), K-IFRS 제·개정, 입법예고/규정변경예고, 중대 회계부정 별도 보도, 심사·감리 지적사례, 회계기준원 질의회신

**요약 X**: 시험 일정·합격자, 단순 통계, 절차적 공지, 포상금 실적, 뉴스레터

**경계 사례**:

- 동일 보도자료 금감원·금융위 양쪽 → 원 발표 기관에 요약, 다른 쪽 링크만
- 동일 보도자료 금감원·기준원 양쪽 → 정책 주체에 요약, 기준원은 별도 관점(제·개정 공표 등) 있을 때만
- 조사·감리결과 금감원·금융위 동일 건 → 금융위에 Type A 표, 금감원 링크만
- 중대 회계부정 별도 보도 → Type A/B 표와 별도 `!!! note "주요 내용"` 서술 블록

## 1.4 제재 조치 표

### Type A — 금융위 최종 과징금 부과

표 스키마 (**변경 금지**): `| 회사명 | 대상자 | 위반내용 | 과징금 부과액 |`

- 금액: **N.N백만원**, 반복 부호(〃) → 원래 텍스트 치환
- 회사명: 변경 시만 기재, 후속 행 비움
- 대상자: `회사`, `대표이사`, `前대표이사 등 ○인`, `담당임원 등 ○인`, 회계법인명
- 위반: `회계처리기준을 위반하여 재무제표 작성` / `회계감사기준을 위반하여 감사업무 수행`

### Type B — 증선위 조치 의결

접기 블록 안에 **회사별 조치 요약** 표는 항상 포함. **감사인 및 공인회계사 조치 요약** 표는 붙임에 감사인·공인회계사 조치가 있을 때만 포함(조치가 없으면 해당 표 생략).

1. **회사별 조치 요약**: `| 회사명 | 구분 | 주요 지적사항 | 주요 조치 |`
2. **감사인 및 공인회계사 조치 요약**(조치가 있을 때만): `| 회사 | 주요 지적사항 | 대상 | 조치 |`

- **구분**: 유가증권시장/코스닥시장 상장, 비상장 등 붙임 기준으로 구체 표기
- **지적사항·조치**: 웹 개요가 아닌 **붙임(PDF/HWP)**에서 구체 위반 내용(연결대상 범위 확대, 매출 과대계상 금액 등)·조치(과징금 금융위 최종 예정, 감사인지정 N년, 손배기금 추가적립 N% 등) 추출
- 한공회 위탁감리위원회 조치 → 회사 열에 `(한공회 위탁감리)`, 지적사항에 위반 유형·대상 회사 수

## 1.5 특수 유형 요약

### 심사·감리 지적사례

`!!! note` 안에 **비불릿 서두 문단**(맥락·통계) + **중첩 `??? info`**(검색경로 링크, ■ 카테고리 헤더, 번호 목록)

### 중대 회계부정 서술 블록

Type A/B 표와 **별도 링크**에 대해 `!!! note "주요 내용"` 작성. 괄호 접두어((조치 개요)·(위반 1)·(유의사항)·(향후))로 서사 구성. 중대성 지표(자기자본 대비 비율, 검찰 고발 등) 포함.

### 입법예고/규정변경예고

외부감사법 관련 개정안 반드시 요약. 괄호 접두어((개정 이유)·(가중·감면)·(의견 제출) 등). 의견 제출 기한 **볼드**.

### 회계기준위원회·지속가능성기준위원회 회의(한국회계기준원 주요일정)

`!!! note "주요 내용"` 안에 `□ 의결 안건` / `□ 보고 안건` 헤더, 각 `- 제○호 …` 불릿. 해당 없으면 `해당사항 없음`.

- **줄 배치**: `□ 의결 안건` 다음 **빈 줄 1개** → 의결 불릿 나열 → **빈 줄 1개** → `□ 보고 안건` → **빈 줄 1개** → 보고 불릿 나열.
- **볼드**: 의결·보고 안건 내 제도명(K-IFRS 제○호, 질의회신 등)에는 볼드 사용하지 않음(일관된 목록 스타일).
- 기준 예시: `2025-10-01_to_2025-12-31.md` 내 회계기준위원회 요약 블록 참조.

# Phase 2: 분기 요약

## 구조 (엄수)

1. **Executive Summary** → 2. **기관별 요약** → 3. **시사점**. 추가 섹션 불가.

## Executive Summary

- **음/슴체**, **3~5문장**, 불릿 금지, **볼드**는 핵심 테마에만
- 감독 기조·규제 우선순위·방향성 신호 중심. 링크·날짜·개별 사건 상세 불포함

## 기관별 요약

```markdown
#### 기관별 요약

!!! success ""

    === "금융감독원"
        - …
    === "금융위원회"
        - …
    === "한국공인회계사회"
        - …
    === "한국회계기준원"
        - …
```

**기관 역할 구분**:

- **금융위 = 정책**: 제도 수립·개정, 증선위·금융위 조치(과징금·제재)
- **금감원 = 집행**: 검사·감리·심사, 지정 통지, 설명회·간담회, 유의사항 안내
- **한공회**: 실무 지침·감사 유의사항
- **한국회계기준원**: K-IFRS 제·개정, 질의회신(일탈회계·지분법 등), IFRS IC 안내, 정착지원 T/F

> 증선위·금융위 조치는 **금융위원회** 탭에만 기재.

# 공통 규칙

## 원문 처리

- **PDF/HWP**: Phase 0 표에 따라 경로 결정 후 추출. 블록 안에 PDF URL 불포함.
- **증선위·금융위 조사·감리 조치 보도자료**: 웹 페이지만으로는 Type B 상세 불충분. **붙임(PDF/HWP)** 반드시 확인.
- **중복 원문**: 원 발표 기관에만 요약, 다른 쪽 링크만.
- **접근 불가**: `<!-- 원문 접근 불가 -->` 주석.
- 원문 재현 금지, URL 블록 내 불포함, 모든 소스를 권위 있는 사실로 취급.

## Appendix A

파일 말미에 크롤러 전체 목록(필터링 전)을 `??? info` 중첩 구조로 배치. 기관별 `??? info` 블록, 카테고리별 **볼드 헤더**, 링크만 나열(요약 없음).

```markdown
## Appendix A. Complete List of Retrieved Items (Unfiltered)

??? info "전체 자료 (전문가 가공 전)"

    ??? info "금융감독원"
        **보도자료**
        - (YY-MM-DD) [제목](URL)
        ...
    ??? info "금융위원회"
        ...
```
